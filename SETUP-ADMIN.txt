RESTAURANT PISTACHE - SECURE ADMIN CRUD PACKAGE
================================================

This package is designed to be copied over the secure core package.

COPY
----
Copy the admin/ folder into the project root and replace the files with the
same names.

A new helper is included:
admin/app/functions/fct-admin-crud.php

REQUIREMENTS
------------
- The secure core package must already be installed.
- PHP 8.1 or newer.
- The admin/uploads folder must exist or be creatable by PHP.
- fct-ui.php from the secure core package must be present.

BEHAVIOUR
---------
- Admin can add, edit and delete dishes and reservations.
- Guest sees the complete dish-management interface but every mutation is
  blocked on the server.
- Visitor messages and reservation details are hidden from Guest accounts.
- Delete actions use POST and CSRF protection.
- Image uploads use MIME validation, a 5 MB limit and random file names.
- Old images are removed only after a successful update.
- Public single-dish pages show active dishes only, while authenticated users
  may preview inactive dishes.

TEST ORDER
----------
1. Log in as Admin.

3. Edit the starter without changing the image.
4. Edit it again with a replacement image.
5. Delete a test dish.
6. Add and edit a reservation.
